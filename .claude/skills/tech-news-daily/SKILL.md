# tech-news-daily

KAIST 경영대학 테크 네트워크 **데일리 테크 브리프** 홈페이지(`/Users/cheil/dev/technews/`)에 매일 한 호분의 데이터를 자동으로 채워 넣고 GitHub에 푸시까지 완료하는 스킬.

## 원칙

이 스킬은 다음 4가지를 한 번에 자동화한다.

1. **큐레이션**: 시간 컨텍스트, 매체 직접 접근, 4차원 평가, 다양성 필터, **지난 7일 중복 제거**로 10건 선정
2. **글쓰기**: 친절한 존댓말 3문장 산문, 자극어/AI 정형어 회피, 달러는 원화 환산 병기
3. **JSON 작성**: 홈페이지가 기대하는 스키마(`data/YYYY-MM-DD.json`)로 저장
4. **퍼블리시**: `git add` + `commit` + `push` 자동 실행

## 응답 톤 (중요)

스킬 실행 중·후에 사용자에게 답할 때는 **친절한 한국어 존댓말**을 사용한다.

- ✅ "어제(5/22 금요일) 보도를 정리해서 발행하겠습니다."
- ✅ "이미 같은 일자 파일이 있어서 기존 것을 백업하고 새로 만들었습니다."
- ✅ "검색이 좀 부족했어요. 4번 카드는 한 번 더 보강해서 다시 작성해 보겠습니다."
- ❌ "Done." / "Completed." / "Finished publishing."
- ❌ 영문 단답·반말·기술 용어만 나열

사용자가 영어로 묻거나 영어 요청을 하면 영어로 답해도 OK. 디폴트는 한국어.

## 사용 시점 (트리거)

사용자가 다음과 같이 말하면 자동으로 작동:
- "어제 테크 뉴스 정리해줘"
- "오늘의 테크 브리프 만들어줘"
- "데일리 브리프 업데이트해줘"
- "이번 주말 핫한 IT 이슈 알려줘"
- "테크 브리프 발행해줘"
- "어제 테크 정리"

## 실행 프로토콜 (11단계)

각 단계의 상세 규칙은 `references/` 와 `personas/` 의 보조 파일을 참조한다. 이 SKILL.md는 단일 진실 공급원(SOT)이며, 보조 파일들이 SKILL.md를 다시 서술하지는 않는다.

### [0] 입력 파싱

- 사용자 발화에서 **기간**을 추출한다. 명시 없으면 "어제".
- 사용자가 페르소나를 명시했는지 확인한다. 명시 없으면 디폴트(카이스트 MBA).

### [1] 시간 컨텍스트 확정 → `references/time-context.md`

- KST(UTC+9) 기준 발행 일자/요일 결정.
- 공휴일 데이터: `data/kr-holidays.json` (2026~2027) 로드.
- **알고리즘**: 오늘 직전 영업일을 거꾸로 찾고, 그 다음 날부터 어제까지를 다루는 기간(`coverage.from` ~ `coverage.to`)으로 잡는다.
- **평일(화~금)**: 어제 단일 일자.
- **월요일**: 금/토/일 3일 묶음.
- **공휴일 직후 영업일**: 휴일 전체 + 그 직전 평일까지 묶음.
- **오늘이 공휴일/토일이면 발행 스킵**, 사용자에게 안내.
- 결과를 JSON `coverage` 필드(`from`, `to`, `label`)로 저장.
- 사용자에게 한 줄 안내 (예: `"오늘(5/26 화)은 5/23~25 부처님오신날 연휴 직후라 5월 23~25일 묶어서 정리하겠습니다."`).

### [2] 페르소나 적용 → `personas/kaist-mba.md`

- 디폴트는 카이스트 MBA 결.
- 한국 비중 4 / 글로벌 6 가중치. 자본 시그널 비중 ↑.

### [3] 후보 풀 수집 → `references/search-strategy.md`

- **매체 직접 접근이 1차** (WebFetch로 매체 메인·카테고리 페이지 fetch). 쿼리 기반 검색은 보강용.
- 카테고리 A(글로벌 자본/딜), B(한국 산업/기업), C(화제 아티클), D(신서비스/오픈소스) 모두 커버.
- **약 50개 후보 풀** 만들기를 목표로. 적으면 보강 검색.
- 한국어 매체 보도가 있으면 `sources` 앞쪽 배치 (없어도 OK).

### [4] 지난 7일 중복 제거 (필수)

- `data/` 폴더에서 **이번 발행 이전 7일 안의 호** 모두 로드 (최근 5~7개 호).
- 각 카드의 `keywords`·`title` 단어를 비교해서 **이미 다룬 사건**은 후보에서 제거.
- 단순 후속·업데이트는 새 사실(숫자·결정·발표)이 추가된 경우만 통과. 단순 재정리·복기는 탈락.
- 예: 5/22 호에 "Claude Opus 4.7 공개"가 있다면 5/26 호에서 같은 사건은 빼되, "Claude Opus 4.7 한국 도입 사례" 같은 새 후속은 OK.

### [5] 큐레이션 → `references/curation-rules.md`

- 4차원 점수(대화 가치, 자본 시그널, 한국 함의, 신선도): 합산 12점 미만 탈락.
- 다양성 슬롯 배분: 글로벌 자본 2~3 / 한국 단독 3~4 / 인수·딜 1 / 신플레이어 1 / 아티클·OSS 1~2 / 표준·생태계 0~1.
- 최신성 필터: 발행일이 지정 기간 밖이면 무조건 탈락. (후속 보도가 기간 내 있으면 OK.)
- 정치/사회 색채 강한 이슈는 제외 (기술 본질 흐림).
- 같은 회사·같은 사건 중복 제거.

### [6] 작성 → `references/writing-style.md`

- 제목: 한글 22~32자, 회사명·핵심 숫자 포함.
- 본문: **정확히 3문장**, "~습니다/입니다" 존댓말 산문, 글머리표 절대 금지.
- 달러는 원화 환산 병기, 큰 숫자는 한국인 체감 단위 비교.
- 금지어(자극어, AI 정형어) 회피.

### [7] 자체 검수 → `references/self-check.md`

- 11개 체크리스트 모두 통과해야 한다.
- 미통과 항목이 있으면 해당 카드를 재작성한 뒤 재검수. 통과할 때까지 반복.

### [8] JSON 파일 작성 → `references/schema.md`

다음 위치에 신 포맷(`sources` 배열)으로 저장:

- 경로: `/Users/cheil/dev/technews/data/<id>.json`
- `id`: 발행 일자 (`YYYY-MM-DD`)
- `volumeNumber`: `data/` 폴더의 최신 일자 파일 안의 `volumeNumber + 1`. 파일이 하나도 없으면 `1`.
- `dayOfWeek`: 한 글자 (`월`, `화`, `수`, `목`, `금`, `토`, `일`).
- `publishedAt`: `<id>T06:30:00+09:00` 형태 ISO.
- `coverage`: `[1]` 단계에서 계산한 `{ from, to, label }`.
- `newsItems`: 정확히 10개. 각 항목은 `order`(1~10), `title`, `summary`(3문장), `sources`(배열) 필드를 가진다.
- 한국어 매체를 `sources` 배열의 **앞쪽**에 배치한다. 영문 원문은 뒤쪽 보조.

**동일 일자 파일이 이미 존재**:
- 사용자에게 덮어쓸지 한 번 묻는다 (`AskUserQuestion`). 발화에 "덮어써", "강제로", "다시 만들어줘" 같은 표현이 있으면 묻지 않고 진행.
- 진행 결정 시 **기존 파일을 백업으로 보존**:
  ```bash
  TS=$(date +%Y%m%d-%H%M%S)
  mv data/<id>.json data/<id>.json.bak-$TS
  ```
- 새 JSON을 정상 위치에 저장. 백업 파일은 git에 추적되지 않도록 `.gitignore`에 `data/*.bak-*` 패턴 권장(이미 없다면 추가).
- 사용자에게 친절히 알린다: "기존 파일을 `data/<id>.json.bak-<타임스탬프>`로 백업하고 새로 작성하겠습니다."

### [9] 카톡 산출물 생성 + 사이트 빌드

두 단계로 실행:

```bash
cd /Users/cheil/dev/technews
# 1) 카톡 카드 PNG + 텍스트 + 사이트 OG 이미지
python3 .claude/skills/tech-news-daily/scripts/generate_message.py data/<id>.json

# 2) 사이트 풀 빌드 (해당 일자 페이지 + 루트 갱신)
python3 .claude/skills/tech-news-daily/scripts/build_site.py
```

산출물:

**로컬 전용** (`.gitignore`에 포함, 카톡 앱 첨부용):
- `Message/<id>/card.png` — 9:16 세로 카드뉴스 (1080×1920)
- `Message/<id>/text.txt` — 카톡 복붙용 텍스트 (제호 + 날짜 + 키워드 + 10개 헤드라인 + 3문장 요약 + URL)

**깃 푸시** (사이트 자체):
- `date/<id>/index.html` — 그 일자 풀 정적 페이지 (OG 메타 + 라우팅 변수 박힘)
- `date/<id>/og.png` — 메신저 미리보기용 카드 PNG (= card.png 사본)
- `index.html` — 최신 호와 동일한 콘텐츠 (루트 진입점)

**공유 URL 패턴**: `https://kaistbiztech.github.io/dailytechbrief/date/<id>/`

`generate_message.py`가 Playwright로 PNG 캡처해서 `Message/<id>/card.png`와 `date/<id>/og.png` 두 경로에 저장한다. `build_site.py`는 `index.html` 안 `<!-- BUILD:META:START --> ... <!-- BUILD:META:END -->` 마커 영역을 그 일자 메타·라우팅 변수로 교체해 일자별 정적 HTML을 생성한다.

생성 실패 시: 사용자에게 명확히 보고하되, 본 [8]의 JSON 파일은 이미 저장됨. 다음 단계로 진행한다.

### [10] Git 자동화

원격 저장소: `https://github.com/kaistbiztech/dailytechbrief.git` (브랜치 `main`).

다음을 순차 실행:

```bash
cd /Users/cheil/dev/technews
git add data/<id>.json date/<id>/ index.html
git commit -m "Add daily brief: <id>"
git push
```

`Message/`와 `planning/`은 `.gitignore`로 빠져 추적되지 않는다.

- 각 단계의 실패는 즉시 사용자에게 보고한다. **파일은 이미 저장된 상태**이므로 사용자가 수동으로 후속 작업을 할 수 있도록 명확히 안내.
- 인증이 실패하면 push만 실패하고 commit은 남는다. 그 상태를 정확히 알린다.

## 절대 규칙 (어기지 말 것)

1. **본문은 정확히 3문장**이어야 한다. 2문장도, 4문장도 안 된다.
2. **글머리표 사용 금지**. 줄글 산문만.
3. **자극어 사용 금지**: "충격", "역대급", "대박", "미쳤다", "엄청난", "굉장한".
4. **AI 정형어 금지**: "결론적으로", "요컨대", "다음과 같이", "~라고 합니다".
5. **달러 표기는 항상 원화 환산 병기**.
6. **JSON 스키마 위반 금지** — 홈페이지가 그대로 렌더링하기 때문에 필드 누락/오타가 곧 사이트 깨짐.
7. **사용자가 명시적으로 멈추라고 하지 않는 한 git push까지 자동 진행**.

## 엣지 케이스

- **`data/`에 파일이 0개**: `volumeNumber = 1`로 시작.
- **후보가 10개 미만**: web_search를 1~2회 추가 시도. 그래도 부족하면 사용자에게 솔직히 보고하고 가능한 만큼만 발행할지 묻는다.
- **공휴일 자동 판별 불가**: 사용자가 발화에서 알려준 공휴일 정보를 우선 사용. 모르면 일반 평일 룰로 진행.
- **이미 푸시된 일자 재발행**: 새 commit으로 덮어쓴다. 사용자가 force push를 요청하지 않는 한 force 옵션은 쓰지 않는다.
- **git이 초기화되지 않은 경우**: 파일 저장만 성공하고 git 단계는 건너뛰면서 사용자에게 안내.
